Most of the time, text you see on pages that aren't part of a page or product description you wrote yourself will be translateable/changeable from the language edit screen. Unless it's "hard coded" which is not what we want.

What is "hard coding"?

When text is explictly written in the HTML, we say that it is hard coded. This means it can only be changed if someone has access to the site's files.

HTML


<h1>Fuzzy boots</h1>
<h3>$139.99</h3>
<p>These boots will keep you warm all winter. Their truly amazing!</p>

Consequences of hard coding: Difficuly to change if you don't know where it is; not everyone will know how to change it; sometimes you can't access it.

How does Liquid help?

Instead of writing out explicity what we want, we just reference an object.

Liquid


<h1>{{ product.title }}</h1>

<p>{{ product.description }}</p>

HTML returned


<h1>Rooster Decor Wood Cheese Board</h1>

<p>This is such a lovely wood grain rooster cheese board.
Great country / rustic decor for the rooster collector.</p>

If you want to change something, you just have to login into Shopify and change the page. No need to go into the theme files.

Exercise #4: Intro to liquid filters

  • You've decided you need to see the product's type listed after the product's description.
  • And you want to see the product type in ALL CAPS.

To accomplish the second task, we'll need to know what liquid filters are available to us.
Filters modify whatever they are attached to, whether numbers, strings, variables or objects

Example


{{ product.object | filter }}

There are two instances of product.title - need to choose the right one.

Types of filters

String Filters: for when the variable type is a string.

Liquid


{{ page.title | upcase }}
{{ 'HOW ARE YOU?' | downcase }}
{{ product.handle | append: '.png' }}
{{ article.author | replace: 'Nathan', 'Bob' | prepend: 'Sir ' }}

Syntax Highlighting

Code highlighting is provided by highlight.js and will be applied to any code wrapped in <pre><code></code></pre>. The syntax highlighting theme files can be found in framework > css > highlightjs-themes. If you wish to change it, update the css link in the <head>. Visit highlight.js to see the full list of supported languages, themes and other usage information.

The default theme for this slide presentation uses a light background because some workshop settings don't allow for the dimming of classroom lights or the projector does not show syntax highlighting on dark backgrounds very well.

For HTML, the left angled bracket, <, needs to be escaped to show. Sometimes, for HTML snippets, the plugin may not register it properly as HTML. If that is the case, add an xml class to the <pre> tag.

Here's a tool to help with the conversion: http://www.htmlescape.net/htmlescape_tool.html
